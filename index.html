  • It can be elastically and transparently expanded without downtime
  • It provides broadcasting to many applications
  • Allows to build systems that are event based

1.0.0

What is Karafka?

  • It is a framework
  • It was designed to simplify Kafka based applications development
  • It allows developers to build "Rails like" apps that consume and produce messages

Why Karafka was developed?

We've needed a tool that would allow us to:

  • build applications faster
  • process faster
  • handle events and messages from many sources and process them the same way

Why Karafka was developed?

Developers should focus on the business logic

But:

  • threads
  • signals
  • connection management
  • load balancing
  • connection failures / reconnections

aren't business logic

HTTP like feeling in Kafka world?

But the truth is, Kafka world is nothing like HTTP

Different scaling strategies

  • Threads are not homogenic (like with Puma or Sidekiq)
  • Their internals can act differently depending on the app settings
  • Multi-process single application deployment support

Non homogenic threads

  • Long living controllers
  • Short living controllers (for both message and message batches)

Different processing strategies

  • Inline processing
  • Sidekiq processing
  • Guild processing (Ruby 3.0)

Params lazy evaluation

params_batch is not just a simple array. The messages inside are lazy parsed upon first usage.

This is super useful when filtering data based on Kafka metadata

Params lazy evaluation


def perform
  params_batch.to_a.each do |param|
    param[:parsed] #=> false
    next unless User.exists?(param[:key])
    # Some heavy parsing happens here
    param.retrieve!
    param[:parsed] #=> true
    User.where(id: param[:key]).update!(param[:user])
  end
end

Karafka as a non Rails Way stack layer

  • Kafka and Karafka can be a great alternative to callbacks and other alternative notifiers (broadcasting instead of callbacks)
  • It can acts as a flow normalization layer for high throughput tasks
  • Due to it's nature it can easily acts as an auto-scaling layer for handling incoming data spikes

Karafka as a non Rails Way stack layer

Because Karafka provides a message transportation and processing layer, it works really great with other modern Ruby solutions including:

  • Reform
  • Trailblazer
  • ROM
  • Dry ecosystem
  • Eventide (for some of the layers)
